# **Context**

Over 70% of students now use large language models (LLMs) during their education, yet current models underperform on chemistry problems when compared to other technical curricula \[1-3\]. This performance gap reflects fundamental limitations in LLMs' approaches to scientific reasoning. These tools are thus failing chemistry students twice: they produce less accurate answers and, even when correct, lack the transparent step-by-step reasoning required for learning.

This stems from unique challenges in handling chemical data, which is diverse, difficult to represent, and rich in specialized symbolic reasoning requirements that general-purpose models struggle to process \[2\]. To address this, we develop a fine-tuned model designed to explicitly map out the mathematical operations, chemical principles, and logical steps employed when solving chemistry problems. Additionally, our lightweight architecture would require minimal computational resources, enabling local deployment without cloud infrastructure. This approach should not only improve educational outcomes broadly but also democratize chemistry education by making expert-level artificial intelligence (AI) tutoring accessible to under-resourced settings. By strengthening students' foundational problem-solving skills in chemistry, we can better prepare them for careers in chemistry-dependent fields including chemical engineering, materials science, pharmaceutical development, and environmental science.

# **Conditions of Success**

For our fine-tuned (pre-trained) model to be considered a success we must achieve; (1) satisfactory semantic alignment with the ground truth text, (2) low relative error for numerical based answers, and (3) an improvement, on these metrics, over the pre-trained (not fine-tuned) variant of the model.

1) For success, a SciBERTScore will have a F1 score of \>0.7 averaged over all test samples, which suggests semantic consistency in this context \[4-6\].

2) For success, the mean relative error (invariant to scale) will be \<5% \[7\] across the test samples.

3) For success, the fine-tuned model will achieve statistically significant improved results on SciBERTScore (Wilcoxon signed-rank test \- scores may not be normally distributed) and relative error (paired t-test) when compared to the non fine-tuned (but pre-trained) model.

For further information on these metrics, please see the Evaluation section.

# **Data**

There will be two main sources (and stages) of data for this project. First, an AI-generated dataset will be used as a proof-of-concept for this problem, and then data obtained from chemistry textbooks will be used to further elucidate this. For the former, we are using the following dataset from HuggingFace \[8\], which contains 20,000 pairs of chemistry problems and their solutions from a collection of 25 topics and subtopics, generated by GPT-4. It is a \`.csv\` file with 4 columns of interest: topic (e.g., organic chemistry, inorganic chemistry, biochemistry), sub-topic (e.g., naming compounds, stereochemistry and chirality, functional groups and their reactions), question (written as a string), and the corresponding answer (written as a string). To guarantee the authenticity of the agent of generation for the latter, we are using established chemistry textbooks \[9, 10\] used for teaching the aforementioned topics at the undergraduate level, and performing data scraping on the textbooks and their respective (human-written and solved) solutions manual to obtain human-written questions and answers. Data pre-processing will be performed to clean the data and transform it in such a way that it is in string form, with chemical structures and molecules being parsed into SMILES strings. # **Evaluation**

The metrics for measuring the model’s ability to solve structured chemical reasoning problems are multi-faceted. As the model must solve the problems step-by-step, a vital condition is that the steps taken are semantically correct. Hence, to evaluate success in this regard, and to acknowledge that there are many equivalent ways to explain how to solve and reason through chemical problems, a semantic alignment score, such as BERTScore \[11\], is presented. The algorithm proposed in BERTScore can be used with a more “chemically” aware embedding function such as SciBERT \[12\], deemed SciBERTScore \[4\]. However, while some dataset samples may have purely explanatory or conceptual answers, other samples have numerical final answers (proceeding step-by-step textual problem solving). Hence, for questions that have final numerical answers, we will use a scale-invariant relative error on numerical samples \[7\] in conjunction with the SciBERTScore. We plan to first evaluate our model using an 80/10/10 proportion split, stratified by chemistry area (e.g., chemical kinetics or chemical thermodynamics) to ensure equal relative frequencies. However, we would also like to test topic generalizability and will thus also use adversarial splits by chemistry topic in a separate analysis.

# **Modeling**

Since our input data is text, we will borrow techniques from the language transformer domain (i.e., large language models (LLMs)). Given the vast amount of data and computation required for pre-training LLMs to understand the nuances of human language, we will fine-tune existing pre-trained LLMs using parameter efficient methods such as QLoRA or LoRA \[13\]. We will start by fine-tuning models in the LLama family \[14\], as they are open source and have a large community support base. More specifically, we will aim to fine-tune Meta-LLama-3-8B, Meta-LLama-2-13B, and Meta-LLama-3-70B models (which may require cloud computing) \[15, 16\]. We will also consider two baselines; (1) the non-fine-tuned (only pretrained) version of the model we aim to train, will serve as a lower baseline to ensure that our fine-tuning is providing value. Our second baseline (2), serving as a somewhat upper baseline, will be the enormous (and black-box) SOTA models such as GPT-5.2 \[17\] or Gemini 3.0 Pro \[18\], evaluated through API calls on the same data is used to test the fine-tuned model. 

# **Stretch Goals**

* Mechanistic interpretability of the embedding space of where certain chemical concepts exist in embedding space.

* Retrieval-Augmented-Generation (RAG) to textbooks to expand capabilities.

* Evaluating and adapting for different failure modes in the chemistry domain.

* Expand the model's ability to handle visual data (multimodal).
